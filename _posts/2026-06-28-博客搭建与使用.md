### 环境搭建
#### 一、环境准备

1.安装依赖
```bash
# 更新软件包列表
sudo apt update
# 安装编译工具和 Ruby 开发库
sudo apt install build-essential ruby-dev
```
2.配置 RubyGems 用户目录
```bash
echo 'export GEM_HOME="$HOME/gems"' >> ~/.bashrc
echo 'export PATH="$HOME/gems/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```
3.安装 Jekyll 和 Bundler‌
```bash 
gem install jekyll bundler
```
#### 二、创建自己的博客项目
1.生成项目框架
```bash
jekyll new my-blog
cd my-blog
```
2.本地预览
```bash
#启动服务
bundle exec jekyll serve
```
访问页面http://localhost:4000查看效果。
#### 三、安装VSCode（可选）
#### 四、日常创作流程
1.写文章‌
在 _posts 目录下创建 年-月-日-标题.md 文件，用 Markdown 撰写内容。

2.本地预览‌
在 VS Code 内置终端中运行 bundle exec jekyll serve，修改文章后浏览器自动刷新。

#### 五、版本控制与上线
1.git初始化
```bash 
#初始化git
git init
```
2.创建.gitignore（若无）
```bash
_site/
.sass-cache/
.jekyll-cache/
.jekyll-metadata
vendor/
```
3.创建github仓库\
仓库名：用户名.github.io;\
选Public公开仓库；\
4.配置认证（个人访问令牌）
有效期自行设置，推荐1-3个月；
```bash
#保存令牌，避免在有效期内重复输入
git config --global credential.helper store
```
5.推送上线
```bash
#添加当前路径
git add .
#提交修改
git commit -m "初始化博客"
#本地分支默认是master，但是仓库是main，修改本地为main，保持一致
git branch -M main
#
git remote add origin https://github.com/用户名/用户名.github.io.git
#推送
git push -u origin main
```
#### 六、后续更新
```bash
git add .
git commit -m "描述修改内容"
git push
```
推送后 GitHub Pages 自动部署，1-2分钟后博客主页更新。
