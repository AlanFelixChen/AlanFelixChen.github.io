嵌入式日志库 spdlog 从入门到最佳实践

一、为什么嵌入式开发者需要 spdlog

在嵌入式开发中，日志是调试和监控的命脉。传统的 `printf` 调试方式存在三大痛点：输出目标单一（只能串口）、格式混乱（时间戳全靠手写）、性能低下（每次调用都阻塞等待）。

spdlog 作为一款纯 C++ 的日志库，完美解决了这些问题：

- Header-only 或编译模式：无需链接额外动态库，直接嵌入项目
- 多 Sink 组合：一份日志同时输出到串口、文件、远程服务器
- 异步日志：后台线程写入，不阻塞业务主循环
- 编译期优化：生产环境可剔除低级别日志，零运行时开销

二、快速上手：三分钟搭建第一个示例

2.1 引入 spdlog

推荐使用 Git 子模块管理，便于版本控制：

```bash
cd your-project
git submodule add https://github.com/gabime/spdlog.git
```

2.2 基础示例

```cpp
include "spdlog/spdlog.h"
include "spdlog/sinks/stdout_color_sinks.h"

int main() {
    auto logger = spdlog::stdout_color_mt("console");
    logger->info("Hello, spdlog!");
    logger->warn("This is a warning");
    logger->error("Something went wrong");
    return 0;
}
```

运行后控制台输出带颜色标记的日志，时间戳和线程 ID 自动添加：

```

[2026-07-05 11:36:14.124] [console] [warning] This is a warning

```

三、核心架构：Logger、Sink、Formatter

spdlog 的设计围绕三大组件展开，理解它们就掌握了 spdlog 的精髓。

3.1 Logger（日志器）

用户直接使用的接口，负责接收日志消息并分发给 Sink。每个 Logger 可绑定多个 Sink，实现一份日志多处输出。

```cpp
// 创建 Logger 并绑定多个 Sink
auto console_sink = std::make_shared<spdlog::sinks::stdout_color_sink_mt>();
auto file_sink = std::make_shared<spdlog::sinks::basic_file_sink_mt>("logs/app.log");
spdlog::sinks_init_list sink_list = { console_sink, file_sink };
auto logger = std::make_shared<spdlog::logger>("multi", sink_list);
```

3.2 Sink（输出目标）

决定日志写到哪里。spdlog 内置了丰富的 Sink：

| Sink 类型 | 用途 | 典型场景 |
|-----------|------|----------|
| `stdout_color_sink_mt` | 带颜色的控制台输出 | 开发调试 |
| `basic_file_sink_mt` | 普通文件写入 | 简单日志存储 |
| `rotating_file_sink_mt` | 按大小轮转的文件 | 嵌入式 Flash 存储 |
| `daily_file_sink_mt` | 按日期轮转的文件 | 服务器日志管理 |

3.3 Formatter（格式化器）

控制日志的输出格式。默认格式为 `[日期 时间.毫秒] [线程] [级别] 消息`，可通过 `set_pattern()` 自定义：

```cpp
logger->set_pattern("%Y-%m-%d %H:%M:%S.%e [%l] %v");
```

四、异步日志：让日志不再拖慢你的设备

4.1 工作原理

嵌入式设备通常只有单核 CPU，同步日志会阻塞主循环。spdlog 的异步模式通过后台线程写入，完美解决这个问题。

异步日志的核心是一个多生产者单消费者队列：

```
主线程（业务逻辑）         后台线程（日志写入）
     │                           │
     ├── spdlog::info() ──→ 队列  │
     ├── spdlog::warn() ──→ 队列  │
     ├── spdlog::error() ──→ 队列  ├── 从队列取日志
     │                           ├── 格式化
     │                           └── 写入文件/串口
```

4.2 队列满时的三种策略

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| `block` | 阻塞等待队列有空位 | 嵌入式设备，不允许日志丢失 |
| `overrun_oldest` | 丢弃最旧的日志 | 内存紧张，允许丢失旧日志 |
| `discard_new` | 丢弃最新的日志 | 允许丢失新日志，保证实时性 |

4.3 完整示例

```cpp
include "spdlog/async.h"

int main() {
    // 初始化线程池：队列大小 2048，1 个后台线程
    spdlog::init_thread_pool(2048, 1);

    auto file_sink = std::make_shared<spdlog::sinks::rotating_file_sink_mt>(
        "logs/async.log", 1024 * 1024 * 5, 5
    );

    auto logger = std::make_shared<spdlog::async_logger>(
        "async_logger", file_sink, spdlog::thread_pool(),
        spdlog::async_overflow_policy::block  // 队列满时阻塞，避免日志丢失
    );

    spdlog::set_default_logger(logger);
    spdlog::info("This log is written asynchronously");
    return 0;
}
```

五、嵌入式最佳实践：生产级日志模板

经过多个示例的迭代，总结出一套可直接用于嵌入式项目的日志模板。

5.1 完整代码

```cpp
include "spdlog/spdlog.h"
include "spdlog/sinks/rotating_file_sink.h"
include "spdlog/sinks/stdout_color_sinks.h"
include "spdlog/async.h"
include <string>

// 日志脱敏：自动过滤密码等敏感信息
std::string desensitize(const std::string& msg) {
    std::string result = msg;
    size_t pos = result.find("password=");
    if (pos != std::string::npos) {
        result.replace(pos + 9, result.length() - pos - 9, "**");
    }
    return result;
}

int main() {
    // 1. 初始化异步线程池
    spdlog::init_thread_pool(2048, 1);

    // 2. 创建控制台和文件 Sink
    auto console_sink = std::make_shared<spdlog::sinks::stdout_color_sink_mt>();
    auto file_sink = std::make_shared<spdlog::sinks::rotating_file_sink_mt>(
        "logs/embedded.log", 1024 * 1024 * 5, 5  // 5MB，保留5个文件
    );

    // 3. 设置不同 Sink 的日志级别
    console_sink->set_level(spdlog::level::warn);  // 控制台只显示警告以上
    file_sink->set_level(spdlog::level::info);     // 文件记录信息以上

    // 4. 创建异步 Logger
    spdlog::sinks_init_list sink_list = { console_sink, file_sink };
    auto logger = std::make_shared<spdlog::async_logger>(
        "embedded", sink_list, spdlog::thread_pool(),
        spdlog::async_overflow_policy::block
    );
    spdlog::set_default_logger(logger);

    // 5. 错误降级：文件写入失败时切换为仅控制台输出
    spdlog::set_error_handler([](const std::string& msg) {
        fprintf(stderr, "spdlog error: %s, falling back to console\n", msg.c_str());
        auto fallback = spdlog::stdout_color_mt("fallback");
        spdlog::set_default_logger(fallback);
    });

    // 6. 按 error 级别自动刷新
    logger->flush_on(spdlog::level::err);

    // 7. 模拟嵌入式设备日志
    spdlog::info("System boot completed");
    spdlog::warn("Battery level low: 15%");
    spdlog::error(desensitize("Login failed: user=admin, password=123456"));
    spdlog::info("Sensor data: temperature=25.3C, humidity=68%");

    // 8. 确保所有日志被刷新并释放资源
    spdlog::shutdown();
    return 0;
}
```

六、进阶技巧：自定义 Sink 与动态路由

6.1 自定义 Sink：写入 SPI Flash

当内置 Sink 无法满足需求时，只需继承 `base_sink` 并实现两个虚函数。以下是一个写入 SPI Flash 的完整示例：

```cpp
include "spdlog/sinks/base_sink.h"
include "spi_flash_driver.h"  // 假设的 SPI Flash 驱动

template<typename Mutex>
class spi_flash_sink : public spdlog::sinks::base_sink<Mutex> {
private:
    uint32_t current_addr_;  // 当前写入地址
    uint32_t max_addr_;      // 最大地址

protected:
    void sink_it_(const spdlog::details::log_msg& msg) override {
        // 1. 格式化日志消息
        spdlog::memory_buf_t formatted;
        spdlog::sinks::base_sink<Mutex>::formatter_->format(msg, formatted);
        std::string log_str = fmt::to_string(formatted);

        // 2. 检查 Flash 空间
        if (current_addr_ + log_str.size() > max_addr_) {
            current_addr_ = 0;  // 循环写入
        }

        // 3. 写入 SPI Flash
        spi_flash_erase_sector(current_addr_);
        spi_flash_write(current_addr_, log_str.c_str(), log_str.size());

        // 4. 更新地址
        current_addr_ += log_str.size();
    }

    void flush_() override {
        // SPI Flash 写入即持久化，无需额外刷新
    }

public:
    spi_flash_sink(uint32_t start_addr, uint32_t max_addr)
        : current_addr_(start_addr), max_addr_(max_addr) {}
};

// 使用示例
auto flash_sink = std::make_shared<spi_flash_sink<std::mutex>>(0x10000, 0x80000);
auto logger = std::make_shared<spdlog::logger>("flash", flash_sink);
logger->info("Sensor data: temperature=25.3C");
```

6.2 动态路由

使用 `dist_sink` 根据日志级别动态分发到不同目标：

```cpp
auto dist_sink = std::make_shared<spdlog::sinks::dist_sink_mt>();
dist_sink->add_sink(console_sink);  // 所有日志都输出到控制台

// 根据日志级别动态添加文件 Sink
auto error_sink = std::make_shared<spdlog::sinks::rotating_file_sink_mt>(
    "logs/error.log", 1024 * 1024, 3
);
dist_sink->set_level(spdlog::level::err);  // 仅 error 级别写入文件
```
